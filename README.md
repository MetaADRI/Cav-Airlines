# Our Web Page - Styling Guide

In this project, we’re using a main CSS file called [ ``` common.css ```](https://github.com/botxplo01/CAV-Zambia-Airlines/blob/main/frontend/css/common.css) to store all the default styles that are shared across the entire website. Inside ``` common.css ```, we define reusable CSS variables under the [``` :root ```](https://github.com/botxplo01/CAV-Zambia-Airlines/blob/main/frontend/css/common.css) selector — like ``` --montserrat-font ``` and ``` --oxygen-font ``` for fonts, and ``` --purple-color ``` and ``` --white-color ``` for colors. These variables help us keep our design consistent and make it easy to make changes in one place instead of repeating the same styles on every page. If we want to use these variables in other CSS files like [``` home.css ```](https://github.com/botxplo01/CAV-Zambia-Airlines/blob/main/frontend/css/home.css) or ``` about.css ```, we just import ``` common.css ``` at the top using ``` @import url('common.css'); ```. Then we can use the variables by writing things like ``` font-family: var(--montserrat-font); ``` or ``` color: var(--purple-color); ```. We're also going to include the default styles for the navigation bar ``` (<nav>) ``` in common.css. That way, the nav bar will have the same font, colors, layout, and spacing on every page without having to copy the styles into each file. Adding the nav bar styling to ``` common.css ``` makes our code cleaner, easier to manage, and ensures the navigation looks the same across the whole site.
